## Context

The `emb` Ruby gem is currently at the repo root under `gem/`. We want a `gems/` directory that can host multiple gems, starting with `gems/emb/`. Each gem needs to be independently testable with `rake`, have its own README, and have its own CI workflow.

## Goals / Non-Goals

**Goals:**
- Move `gem/` to `gems/emb/` without breaking paths
- `rake` in `gems/emb/` runs the RSpec test suite
- README documents how to run E2E tests (start emb, run `rake`)
- Standalone CI for the gem builds `emb` and runs `rake`
- flake.nix exposes Ruby for `gems/emb/` development

**Non-Goals:**
- Changing the gem's API or implementation
- Publishing to rubygems.org
- Creating a second gem (just the structure for it)

## Decisions

### Decision: Directory layout

```
gems/
  emb/
    .github/workflows/test.yml
    .gitignore
    Gemfile
    Gemfile.lock
    Rakefile
    README.md
    VERSION
    emb.gemspec
    lib/
      emb.rb
      emb/
        client.rb
        multi.rb
        proxy.rb
        version.rb
    spec/
      spec_helper.rb
      emb_spec.rb
    vendor/bundle/
```

### Decision: Rakefile

```ruby
require "rspec/core/rake_task"
RSpec::Core::RakeTask.new(:spec)
task default: :spec
```

Running `rake` from `gems/emb/` runs all specs.

### Decision: README structure

The README covers:
1. What the gem is
2. Prerequisites (emb binary, nix develop or manual ORT setup)
3. Setup: `nix develop` from repo root, or `bundle install` + build `emb`
4. Running tests: `bundle exec rake` or `rake`
5. Example usage: `Emb[:model]["text"]`

### Decision: Standalone CI

The gem's CI at `gems/emb/.github/workflows/test.yml`:
```yaml
jobs:
  test:
    steps:
      - checkout repo
      - Source versions
      - Install ORT + libtokenizers + Go
      - Build emb binary
      - Start emb server
      - cd gems/emb && bundle exec rake
```

### Decision: Flake integration

`nix develop` from repo root already provides `ruby_3_4` and `bundler` (added in previous change). From within `gems/emb/`, user runs `bundle exec rake`. No additional nix changes needed — the dev shell covers both Go and Ruby.

### Decision: Path updates

- `emb.gemspec`: `lib/` paths stay relative, no change needed
- `spec_helper.rb`: `$LOAD_PATH.unshift` stays relative, no change needed
- `.bundle/config`: regenerated by `bundle install` at new location
- `vendor/bundle/`: moved with the rest, `bundle exec` still works

## Risks / Trade-offs

- **CI duplicates** — the gem CI duplicates some setup steps (installing ORT, building emb) that the main CI already does. Mitigation: necessary for independent gem testing; the `Source versions` step keeps versions in sync.
- **Flake.nix only covers root** — user must run `nix develop` from repo root, then `cd gems/emb/`. Mitigation: add a shell.nix or devShell per gem if needed later.
- **vendor/bundle/ is large** — committed vendor dir. Mitigation: add `gems/*/vendor/` to `.gitignore` and let each gem manage its own vendor.
